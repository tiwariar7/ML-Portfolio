About the Project



This project aims to predict the likelihood of a patient having a certain disease based on their symptoms. Using a dataset sourced from Kaggle, the model analyzes a comprehensive list of symptoms, such as itching, skin rash, continuous sneezing, joint pain, high fever, and fatigue, among others. By leveraging advanced Machine Learning algorithms, the application identifies patterns in the data to provide accurate predictions of diseases, including but not limited to fungal infections, allergies, diabetes, and bronchial asthma.
Our goal is to enhance early detection and medical decision-making by providing a reliable, symptom-based diagnostic tool. This tool can be particularly beneficial for preliminary screenings, guiding users to seek appropriate medical attention promptly.



Problem Statement of Project

Develop an advanced machine learning solution to accurately predict human diseases based on reported symptoms. This solution will leverage multiple machine learning models to enhance prediction reliability and ensure robust inference. By integrating diverse algorithms, the approach aims to optimize diagnostic precision and provide seamless, user-friendly predictions in JSON format for real-world applicability. The model's performance will be rigorously evaluated using proven metrics to ensure high-quality results.



Approach Used:


Conceived Data Preparation:


Gathered and engineered a dataset from Kaggle, comprising two CSV filesâ€”one for training and another for testing. Integrated 133 columns, where 132 represent symptoms and the final column denotes prognosis.


Crafted Data Cleaning Process:


Diagnosed data quality and engineered transformations to ensure precision. The prognosis column, originally in string format, was customized into numerical values using a label encoder to enable model compatibility.


Developed Machine Learning Models:


Formulated a plan to train Support Vector Classifier, Naive Bayes Classifier, and Random Forest Classifier. Innovated a method using a confusion matrix to forecast model performance effectively.


Engineered Prediction Pipeline:


Devised a strategy to combine predictions from all three models, enhancing robustness and accuracy. Crafted a user-friendly function that takes symptom input, diagnoses disease using trained models, and outputs predictions in JSON format.


Implementation Guidelines:


Planned the workflow: Download the train and test datasets, place them in the dataset folder, and execute code cells in Jupyter Notebook for clarity and visualized results.





Conceptualized Dataset Utilization:


The dataset is divided into three subsets: Training Data, Test Data, and Validation Data to ensure a well-structured and effective approach.


Engineered Data Preprocessing:


The Training Data (80%) undergoes preprocessing and is split to prepare for model training and evaluation.


Innovated Model Selection Process:


K-Fold Cross-Validation is performed on the training data to identify the most effective model.


Devised Multi-Model Training:


Three machine learning models are trained in parallel: Support Vector Machine (SVM) Classifier, Naive Bayes Classifier, and Random Forest Classifier.


Customized Metrics Computation:


The Test Data (20%) is used to compute performance metrics for each trained model, ensuring precise evaluation of individual models.


Forecasted Predictions on Validation Data:


Each trained classifier predicts outcomes on the Validation Dataset. Predictions from all three models are compared and aggregated.


Integrated Final Prediction:


The mode of all predictions is computed, combining results to generate the Final Prediction, enhancing accuracy and reliability.
